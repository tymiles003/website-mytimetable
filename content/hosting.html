Hosting

Personal timetables

All users have their own personal profile, which enables them to fully personalise their timetable. Depending on the wishes of the institution, students and staff members can add and remove courses, modules, study programmes or other timetables.

MyTimetable can also be integrated with your student information system. This way, the courses students are enroled in are automatically added to their profile.

Views for every use case

Students and staff members will be familiar with and appreciate the graphical views, which display the schedules per day, week, or month. The list view even allows displaying the schedule for every other date range defined in the timetable datasource.

Room availability view

Porters and building managers can use the availability view to quickly locate unoccupied locations in a building.

Synchronisation with third-party calendars

User schedules can be published to Microsoft Exchange or Google Calendar using push synchronisation, providing up-to-date staff availability information.

Other applications can be linked to MyTimetable by using the iCalendar feeds. Microsoft Outlook, Google Calendar, Zimbra and many other applications support this format and are able to keep a users timetable up-to-date continuously.

Mobile ready

A clean and lightweight mobile interface is automatically shown when using a mobile phone. MyTimetable runs in your mobile browser, so there's no need to install an app.

Exports in many formats

Users can export their schedule in multiple formats, such as PDF, CSV, iCalendar and xCalendar. This gives users the opportunity to use their timetabling data in various applications (e.g., Microsoft Outlook, Excel, Google Calendar, Apple Calendar, and more). The PDF option makes it easy to print your timetable while retaining the great layout of MyTimetable.

Open API

A fully RESTful API is available to reuse timetabling data and user profile data in other applications. Data is returned in JSON or XML formats. The API is secured using the OAuth 2 protocol, putting the end-users in control of their data.

Security

Timetabling data can be fully secured using a role-based access model. For example, staff or draft timetables can be hidden from students when necessary, or students can be locked down to their personal timetable.
